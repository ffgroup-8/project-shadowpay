# Project Tracker Rules

When working on index.html for PayNearMe:

- Always read the existing index.html before making changes
- Never remove or downgrade any status pill already marked completed or in-progress
- Update "Last updated" date on every change
- After any change: git add index.html && git commit -m "[description of change]" && git push
- Progress bar data-target values must match the actual % of completed pills per category
- Rings chart animates from data-target values — always keep in sync with progress bars
- Section order: 1. Sitemap Tree → 2. SEO/AEO Pages → 3. Page Content Strategy → 4. Knowledge Base
