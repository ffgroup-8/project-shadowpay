How This Tracker Works:

1. Status updates: When a page category is completed, tell Claude which page and which category (Design, Dev, SEO, Content, Sanity, Created) to mark as complete. Claude will update the pill from ○ Not Started → ✓ Completed and recalculate progress bars.

2. Git workflow: After each update — git add index.html && git commit -m "..." && git push — so the client always sees current status on GitHub Pages.

3. SEO/AEO pages: Add landing pages to Section 2 as keyword research identifies opportunities.

4. Blog articles: Individual posts not tracked here — managed through Sanity. The /blog listing page IS tracked.

5. Knowledge base: Section 4 is the single source of truth for all content creation. Update as client docs arrive.

6. Content section structure: Green SEO box = metadata only. Blue Content Created box = all 📌 Module copy once written and approved. Purpose sits above both boxes as plain text. Never nest or put module content outside the blue box.
